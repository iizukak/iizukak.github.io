Docker で MPI は使いづらい？

2018/09/03

Tags: dialy

Green500 のサンプルコードを実行したいと思っているのだが、MPI が使われていてこれが Docker では動かないらしい。HPC 用のコンテナ技術として、Singurality というのがあるようで、こちらだと MPI やらなにやら HPC 関連に特化したコンテナになっているようだ。使い勝手がよければ使ってみても良い。

一瞬考えた計画としては

  • 手元のコンテナで Graph500 リファレンス実装を動かす　
  • AWS の弱いインスタンスでリファレンス実装を動かす
  • AWS の強いインスタンス(スポットインスタンス)でリファレンス実装を動かす
    • ここまではリファレンス実装でいける想定
  • AWS のノードをいくつか使ってベンチマークをとる
    • リファレンス実装のままでいけるのか？
  • AWS の GPU インスタンスでベンチマーク
  • AWS の FPGA インスタンスでベンチマーク
    • Xilinx SDAccell あたりを使う

とりあえず Singurality を使ってみるか。だめだったら、Docker で MPI を使う方法を探っていくことにする。

仕事はバージョンコントロールシステムの評価。ぼちぼちの進捗。
